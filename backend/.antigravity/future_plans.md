# Планы на будущее (Backend Roadmap)

## Оптимизация и Кэширование
- [ ] **Redis Feed**: Переход на хранение ID постов для ленты в Redis (паттерн "Redis Feed").
- [ ] **Персональные рекомендации**: Реализация набора ID постов для каждого пользователя в Redis.
- [ ] **Кэширование комментариев**: Хранение ID комментариев в Redis для мгновенного доступа.
- [ ] **Rate Limiting**: Ограничение количества запросов (лайков/постов) в минуту для экономии ресурсов.

## Архитектура
- [ ] **Short Serializers**: Внедрение `UserShortSerializer` во всех списках для уменьшения JSON payload.
- [ ] **Оптимизация базы комментариев**: Пагинация комментариев (CursorPagination) и хранение метадаты в Redis, чтобы большие треды под виральными постами не вешали базу.
- [ ] **Популярные теги**: Реализовать подсчет использования тегов (через поле `usage_count` или агрегацию) и API для вывода самых популярных тегов на главном экране.
- [ ] **3-х уровневая фильтрация тегов**: На фронтенде и бэкенде реализовать последовательный выбор: Теги Ресторана -> Теги Блюда -> Теги Поста для максимально точного поиска.
- [ ] **Media Optimization (Celery)**: Запуск задач на сжатие, ресайз и конвертацию загружаемых изображений в WebP для экономии места и ускорения приложения по сети.

# Foody 🍔

Проект для поиска и рекомендации еды, развернутый в Yandex Cloud.

## 🛠 Локальная разработка

Вы можете продолжать разработку локально, используя облачную базу данных и хранилище.

1.  **Установка зависимостей**:
    ```bash
    npm install
    ```

2.  **Запуск сервера разработки**:
    ```bash
    npm run dev
    ```
    Сайт будет доступен по адресу: `http://localhost:3000`

> [!NOTE]
> Локальный сервер использует настройки из файла `.env`. Сейчас он подключен к облачной базе PostgreSQL в Yandex Cloud.

---

## 🚀 Цикл обновления (Workflow)

Когда вы внесли изменения и проверили их локально, следуйте этой инструкции, чтобы обновить сайт в облаке:

### 1. Изменение базы данных (если нужно)
Если вы изменили файл `prisma/schema.prisma`, синхронизируйте базу:
```bash
npx prisma db push
```

### 2. Сборка нового образа
Соберите новую версию Docker-образа. Рекомендуется увеличивать номер версии (v5, v6...):
```bash
docker build -t cr.yandex/[ID_ВАШЕГО_РЕЕСТРА]/foody:v5 .
```

### 3. Отправка в облако
```bash
docker push cr.yandex/[ID_ВАШЕГО_РЕЕСТРА]/foody:v5
```

### 4. Обновление в Yandex Cloud
1.  Зайдите в консоль **Serverless Containers**.
2.  Выберите контейнер `foody-app`.
3.  Нажмите **Создать ревизию**.
4.  Выберите новый тег образа (например, `v5`).
5.  Нажмите **Создать ревизию**.

Через несколько секунд изменения вступят в силу на живом сайте!

## 🗂 Структура проекта
- `/src/app`: Страницы и логика приложения (Next.js App Router).
- `/src/components`: UI компоненты.
- `/src/lib`: Утилиты (подключение к БД, облачное хранилище).
- `/prisma`: Схема базы данных.
